# 个人技术展示网站产品需求文档

## 1. 产品概览

个人技术展示网站是一个专为面试场景设计的个人品牌展示平台，旨在通过可视化方式全面展示候选人的技术栈、专业成就和项目经验。

- 该网站解决了传统简历信息展示有限、无法直观呈现技术能力的问题，帮助候选人在面试过程中脱颖而出。
- 网站核心价值在于通过专业、美观的界面设计，将个人技术能力和成就转化为直观、可量化的视觉呈现，提升面试竞争力。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 访问者（面试官） | 无需注册 | 浏览所有公开内容，查看技术栈、奖项、项目经验等信息 |
| 网站所有者 | 直接管理代码 | 编辑网站内容，更新技术栈、奖项和项目信息 |

### 2.2 功能模块

个人技术展示网站包含以下核心页面：
1. **首页**：个人简介、核心技术栈概览、联系方式
2. **技术栈页面**：详细技术分类展示、技能熟练度可视化
3. **奖项页面**：奖项列表、详细描述、获奖时间线
4. **项目经验页面**：项目列表、技术栈应用、成果展示
5. **关于我页面**：个人背景、职业目标、教育经历

| 页面名称 | 模块名称 | 功能描述 |
|---------|----------|----------|

| 首页 | 最近奖项 | 展示最近获得的2-3个重要奖项，点击可跳转到奖项详情页 |

| 技术栈页面 | 技术分类 | 按前端、后端、数据库、工具等分类展示技术栈 |
| 技术栈页面 | 技能熟练度 | 使用进度条或雷达图可视化技能熟练度（初级、中级、高级、专家） |

| 奖项页面 | 奖项详情 | 点击奖项可展开详细描述，包括获奖原因、成果展示等 |
| 奖项页面 | 时间线 | 以时间线形式展示获奖历程，突出职业成长轨迹 |
| 项目经验页面 | 项目列表 | 展示个人参与或主导的项目，包含项目名称、时间、角色 |
| 项目经验页面 | 项目详情 | 点击项目可展开详细描述，包括项目背景、技术栈应用、解决的问题、取得的成果 |
| 项目经验页面 | 项目链接 | 提供项目演示链接、GitHub仓库链接等（如果有） |
| 关于我页面 | 个人背景 | 详细介绍个人职业经历、专业背景、核心竞争力 |
| 关于我页面 | 职业目标 | 阐述短期和长期职业目标，展示个人发展规划 |
| 关于我页面 | 教育经历 | 展示学历背景、专业、毕业时间、相关课程等 |
| 关于我页面 | 兴趣爱好 | 简要介绍个人兴趣爱好，展示个人多元化特质 |

## 3. 核心流程

### 3.1 访问者（面试官）浏览流程

1. 访问网站首页，了解候选人基本信息和核心技术栈
2. 浏览技术栈页面，详细了解候选人的技术能力和熟练度
3. 查看奖项页面，了解候选人的专业成就和荣誉
4. 浏览项目经验页面，了解候选人的实际项目经验和技术应用能力
5. 查看关于我页面，了解候选人的背景和职业目标
6. 通过联系方式与候选人取得联系

### 3.2 网站所有者管理流程

1. 克隆网站代码到本地开发环境
2. 更新个人信息、技术栈、奖项和项目经验等内容
3. 测试网站功能和响应式布局
4. 部署更新后的网站到托管平台
5. 验证部署结果，确保网站正常运行

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深蓝色（#165DFF）作为主色调，象征专业、可靠
- **辅助色**：浅灰色（#F5F7FA）作为背景色，白色（#FFFFFF）作为卡片背景
- **强调色**：橙色（#FF7D00）用于突出重要信息和按钮
- **按钮风格**：圆角矩形，主按钮使用主色调填充，悬停时有轻微阴影和颜色变化
- **字体**：使用无衬线字体（如Inter、Roboto），确保跨平台一致性
- **字体大小**：标题24-36px，副标题18-24px，正文14-16px，小文本12px
- **布局风格**：卡片式布局，清晰的视觉层次，适当的留白
- **图标风格**：使用Font Awesome图标库，保持风格统一

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|----------|--------|
| 首页 | 导航栏 | 固定顶部，背景色为白色，阴影效果，响应式设计（移动端变为汉堡菜单） |
| 首页 | 个人简介区 | 居中布局，个人照片圆形裁剪，姓名使用大号字体，核心优势标签使用圆角矩形背景 |
| 首页 | 技术栈概览 | 网格布局，每个技术栈使用图标+文字组合，图标使用对应技术的官方logo或风格统一的图标 |
| 首页 | 最近奖项 | 卡片式布局，包含奖项名称、颁发机构、时间，卡片有轻微阴影和悬停效果 |
| 技术栈页面 | 技术分类 | 标签式导航，点击切换不同技术分类，分类下使用网格布局展示具体技术 |
| 技术栈页面 | 技能熟练度 | 使用进度条或雷达图可视化，进度条使用主色调填充，雷达图使用渐变色 |
| 奖项页面 | 时间线 | 垂直时间线设计，每个奖项节点包含日期、奖项名称、颁发机构，使用连接线连接 |
| 项目经验页面 | 项目列表 | 卡片式布局，每个卡片包含项目名称、时间、技术栈标签、项目描述，悬停时有轻微放大效果 |

### 4.3 响应式设计

- **设计理念**：采用移动优先的响应式设计理念，确保在各种设备上都能良好展示
- **断点设置**：
  - 移动端：< 768px
  - 平板：768px - 1024px
  - 桌面：> 1024px
- **适配策略**：
  - 移动端：单列布局，导航栏变为汉堡菜单，卡片垂直排列
  - 平板：双列布局，导航栏保持简化版，卡片网格布局
  - 桌面：多列布局，完整导航栏，卡片网格布局，增加侧边栏（如果需要）
- **交互优化**：
  - 移动端：增大点击区域，优化触摸操作
  - 平板/桌面：增加悬停效果，优化鼠标操作

## 5. 技术实现方案

### 5.1 技术选型

| 技术类别 | 技术名称 | 版本 | 选型理由 |
|---------|---------|------|----------|
| 前端框架 | 纯HTML5/CSS3/JavaScript | - | 轻量级，无需复杂框架，加载速度快，易于维护 |
| CSS框架 | Tailwind CSS | 3.x | 实用优先的CSS框架，提供丰富的工具类，加速开发，响应式设计支持良好 |
| 图标库 | Font Awesome | 6.x | 丰富的图标资源，支持技术栈、社交平台等各种图标需求 |
| 图表库 | Chart.js | 4.x | 轻量级图表库，用于技能熟练度可视化和数据展示 |
| 动画效果 | AOS (Animate On Scroll) | 2.x | 滚动触发动画，提升用户体验，使页面更具活力 |
| 部署平台 | GitHub Pages | - | 免费托管，与代码版本控制集成，部署简单，适合个人项目 |
| 域名服务 | 自定义域名或GitHub Pages子域名 | - | 根据个人需求选择，自定义域名更专业 |

### 5.2 页面结构设计

```
├── index.html          # 首页
├── skills.html         # 技术栈页面
├── awards.html         # 奖项页面
├── projects.html       # 项目经验页面
├── about.html          # 关于我页面
├── css/
│   └── style.css       # 自定义样式
├── js/
│   ├── main.js         # 主要JavaScript逻辑
│   └── charts.js       # 图表相关逻辑
├── assets/
│   ├── images/         # 图片资源
│   └── icons/          # 图标资源
└── README.md           # 项目说明
```

### 5.3 关键功能实现方案

1. **技术栈展示**：
   - 使用网格布局展示技术分类
   - 使用Chart.js实现技能熟练度雷达图
   - 技术标签使用对应技术的官方颜色和图标

2. **奖项展示**：
   - 使用垂直时间线组件展示获奖历程
   - 卡片式布局展示奖项详情
   - 按时间倒序排列，突出最新成就

3. **项目经验展示**：
   - 卡片式布局，包含项目名称、技术栈、成果
   - 悬停效果展示更多项目详情
   - 技术栈标签使用不同颜色区分技术类别

4. **响应式设计**：
   - 使用Tailwind CSS的响应式工具类
   - 媒体查询适配不同屏幕尺寸
   - 移动端导航栏使用汉堡菜单

5. **性能优化**：
   - 图片懒加载
   - 代码压缩
   - 图标使用SVG或字体图标
   - 减少HTTP请求

## 6. 部署和维护计划

### 6.1 部署流程

1. **代码版本控制**：使用Git进行代码管理，提交到GitHub仓库
2. **构建过程**：本地开发测试，确保代码无误
3. **部署方式**：
   - GitHub Pages：直接从GitHub仓库部署，选择main分支作为源
   - 自定义域名：在GitHub Pages设置中绑定自定义域名，配置DNS解析
4. **部署验证**：访问部署后的网站，验证所有功能正常，响应式布局适配良好

### 6.2 维护计划

1. **内容更新**：
   - 定期更新技术栈，添加新掌握的技能
   - 及时添加新获得的奖项和证书
   - 更新项目经验，添加最新完成的项目
2. **技术维护**：
   - 定期检查网站性能和加载速度
   - 更新依赖库版本，确保安全性
   - 修复可能出现的兼容性问题
3. **备份策略**：
   - 利用GitHub版本控制作为代码备份
   - 定期导出网站数据，保存为本地备份

## 7. 开发计划和里程碑

| 阶段 | 时间周期 | 任务内容 |
|------|----------|----------|
| 准备阶段 | 1天 | 确定网站结构，收集个人信息、技术栈、奖项和项目数据 |
| 开发阶段 | 3-5天 | 搭建网站框架，实现页面布局和基本功能 |
| 内容填充阶段 | 2-3天 | 填充个人信息、技术栈、奖项和项目数据 |
| 测试阶段 | 1-2天 | 测试响应式布局，优化性能，修复问题 |
| 部署阶段 | 1天 | 部署网站到GitHub Pages，配置自定义域名（可选） |
| 优化阶段 | 持续 | 根据反馈调整内容和设计，定期更新网站信息 |

## 8. 风险评估

| 风险 | 影响程度 | 应对措施 |
|------|----------|----------|
| 内容更新不及时 | 中 | 建立定期更新机制，设置提醒，确保技术栈和奖项信息及时更新 |
| 网站加载速度慢 | 中 | 优化图片大小，使用CDN，减少不必要的JavaScript |
| 兼容性问题 | 低 | 测试主流浏览器，使用现代CSS特性的降级方案 |
| 托管平台限制 | 低 | 选择可靠的托管平台，如GitHub Pages，确保稳定运行 |

## 9. 总结

个人技术展示网站是一个专为面试场景设计的个人品牌展示平台，通过可视化方式全面展示候选人的技术能力、专业成就和项目经验。网站采用现代前端技术栈构建，具有响应式设计，确保在各种设备上都能良好展示。

该网站的核心价值在于帮助候选人在面试过程中脱颖而出，通过专业、美观的界面设计，将个人技术能力和成就转化为直观、可量化的视觉呈现，提升面试竞争力。

通过定期更新内容和维护，个人技术展示网站可以成为候选人职业发展的有力工具，为职业规划和求职过程提供持续支持。
